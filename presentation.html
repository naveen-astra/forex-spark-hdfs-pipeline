Introduction
Background
  • The foreign exchange (forex) market is the largest financial market globally, with a daily trading volume exceeding $7.5 trillion.
  • Forex data is high-frequency, multi-dimensional, and arrives in massive volumes — making it an ideal candidate for big data processing.
  • Traditional tools (Excel, pandas) struggle with multi-gigabyte datasets across multiple currency pairs and timeframes.
  • Distributed computing frameworks like Apache Spark can handle this scale while enabling feature engineering, machine learning, and real-time analytics in a unified platform.
  • This project processes 8 years of historical forex data (2017–2024) across 15 currency pairs and 3 timeframes using Spark, Kafka, and Scala.
02 / 15
Problem Statement
Why this project?
  • Forex datasets spanning multiple years, pairs, and granularities result in gigabytes of raw data that cannot be efficiently processed with single-machine tools.
  • Feature engineering (moving averages, volatility, RSI components) requires window-based computations that are computationally expensive at scale.
  • ML model training on large feature sets demands distributed computation to avoid memory bottlenecks and long training times.
  • There is no real-time pipeline to consume live forex data, compute trading signals, and present actionable insights with low latency.
  • A unified system that handles batch processing, ML prediction, and streaming analytics on forex data using big data tools is needed.
03 / 15
Objectives
Project goals
  • Build a Spark-based batch pipeline to preprocess 4.4 GB of raw forex data across 45 CSV files, applying window functions for technical indicator computation.
  • Engineer 19 features including moving averages (MA5, MA20), volatility, lag features, and RSI components using Spark DataFrame API.
  • Train and evaluate Spark MLlib models — Random Forest (24 models), Gradient Boosted Trees (24 models), and Logistic Regression — to predict close price at each of the next 24 hours.
  • Implement a real-time streaming pipeline using Apache Kafka (KRaft mode) and Spark Structured Streaming to generate live trading signals.
  • Deploy windowed aggregations (5-minute sliding windows) for real-time statistical analysis per currency pair.
04 / 15
Related Work
Existing systems & approaches
  • Traditional ML approaches — ARIMA, GARCH, and LSTM models trained with Python/pandas on single-pair data. Limited to small datasets and no real-time capability.
  • Hadoop MapReduce pipelines — Used for forex batch processing but lack interactive querying, iterative ML training, and real-time streaming support.
  • Cloud-based platforms — AWS EMR, GCP Dataproc for distributed processing. High cost and requires cloud infrastructure; not suitable for local development and learning.
  • Kafka + Flink / Storm — Stream processing alternatives. Spark Structured Streaming provides a unified batch + streaming API, reducing system complexity.
  • This project’s advantage — Unified Spark ecosystem for batch, ML, and streaming in a single codebase. Runs locally with no cloud dependency. Uses Kafka KRaft (no Zookeeper/Docker overhead).
05 / 15
Proposed System
Solution overview
  • Batch Layer — SparkHDFSPreprocess.scala reads 45 raw CSVs, applies Window functions (MA5, MA20, Volatility5, RSI components), outputs enriched data to spark_output/.
  • ML Layer — SparkMLModels.scala trains 24 RF + 24 GBT models (one per hour h+1 to h+24) plus LR for 24h direction, using Spark MLlib Pipeline API on hourly data with 19 features.
  • Streaming Layer — ForexKafkaProducer.scala publishes processed rows to Kafka topic at 200ms intervals. SparkKafkaStreaming.scala consumes and generates live BUY/SELL/HOLD signals.
  • Recursive Prediction — ForexKafkaProducer triggers last-row per pair → RecursivePredictor.scala loads saved RF model and runs 24-step recursive loop, publishing hourly forecasts to forex-predictions-live.
  • MongoDB Pipeline — StreamPredictions.scala streams ML output to Kafka (forex-predictions topic). MongoDBSink.scala consumes and batch-inserts 96,294 predictions into MongoDB (forex_db).
  • Live Dashboard — Chart.js-powered dashboard.html renders 24-hour price forecast curves with animated point-by-point display, stats panel, and prediction log.

Methodology
Workflow
Phase 1
Data Ingestion
Read 45 CSV files (15 pairs × 3 timeframes) using spark.read.csv() with schema inference. 4.4 GB raw OHLCV data.
Phase 2
Feature Engineering
Compute MA5, MA20, Volatility5 via Spark Window functions. Add Range, Change, ChangePct, Gain, Loss, lag features.
Phase 3
ML Training
Build Spark ML Pipeline: VectorAssembler → StandardScaler → Model. Train 24 RF + 24 GBT models (one per hour) + LR classifier on 80/20 split using hourly data. 19 features.
Phase 4
Kafka Streaming
Publish processed data to Kafka (KRaft). Spark Structured Streaming consumes, enriches with signals, outputs to console + memory sink.
Phase 5
Recursive Prediction
Trigger last row per pair → Load saved RF model → 24-step recursive loop predicting next hour, feeding output back as input. Publishes to forex-predictions-live.
Phase 6
Live Dashboard
Chart.js dashboard renders 24h forecast curve with animated point-by-point display, stats panel (direction, change %, MA5, volatility), and prediction log.
  • Spark Config — local[*] mode, 4 GB driver memory, AQE enabled, DataFrame API with Window functions.
  • Kafka Config — KRaft mode (no Zookeeper), 3 partitions, localhost:9092, topic: forex-live.

System Modules
Implementation

SparkHDFSPreprocess.scala

Batch preprocessing module. Reads 45 raw CSVs, applies Spark Window functions for MA5, MA20, Volatility5. Computes Range, Change, Gain, Loss. Outputs 10.8 GB enriched data.

SparkMLModels.scala

24-hour prediction module using Spark MLlib Pipeline API. Trains 24 Random Forest + 24 GBT regressors (h+1 to h+24) plus Logistic Regression for direction. VectorAssembler + StandardScaler on 19 features from hourly data.

ForexKafkaProducer.scala

Kafka producer module. Reads preprocessed CSVs row-by-row and publishes JSON messages to topic forex-live at 200ms intervals. Simulates live feed.

SparkKafkaStreaming.scala

Streaming consumer. Reads from Kafka, parses JSON, enriches with MA crossover signals, momentum, volatility alerts. Runs 3 concurrent streaming queries.

RecursivePredictor.scala

24-hour recursive forecasting engine. Loads saved RF h+1 PipelineModel, consumes triggers from Kafka, runs 24-step recursive loop publishing each prediction to forex-predictions-live.

MongoDBSink.scala

Kafka consumer that reads from forex-predictions topic and batch-inserts predictions into MongoDB (forex_db). Stores 96,294 documents across 3 collections.

dashboard.html

Chart.js-powered live dashboard. Renders 24h price forecast curve with animated display, stats panel (direction, change %, MA5, volatility), and scrollable prediction log.

Advantages
Benefits of this system
  • Unified Platform — Single Spark ecosystem handles batch processing, ML training, and real-time streaming. No need for separate tools.
  • Scalability — Spark's distributed DataFrame API scales from local mode to multi-node clusters without code changes.
  • Real-Time Signals — Kafka + Structured Streaming delivers live BUY/SELL/HOLD trading signals with sub-second latency.
  • No Cloud Dependency — Entire stack runs locally. Kafka KRaft mode eliminates Zookeeper and Docker overhead.
  • Fault Tolerance — Spark Structured Streaming provides exactly-once processing with checkpoint-based recovery.
  • Multi-Horizon Prediction — 24 separate models per algorithm provide granular hourly forecasts, showing how accuracy changes over time.
  • Recursive Autoregressive Forecasting — Saved RF model drives a 24-step recursive loop where each prediction feeds back as input, enabling real-time 24h forecasts from a single trigger.
  • Live Visualization — Chart.js dashboard animates 24h forecast curves point-by-point, providing instant visual insight into predicted price trajectories.
  • Reproducibility — All code in Scala with scala-cli directives. Single command to compile and run each module.

Conclusion
Summary
  • Successfully built an end-to-end big data pipeline for forex analysis using Apache Spark, Kafka, and Scala.
  • Preprocessed 4.4 GB of raw data (45 files, 15 pairs, 3 timeframes) into 10.8 GB of feature-enriched output using Spark Window functions.
  • Trained 24 Random Forest + 24 GBT models for multi-horizon prediction — forecasting close price at each of the next 24 hours (h+1 to h+24).
  • Logistic Regression classifies 24h-ahead direction (BUY / SELL), providing trading signals alongside the hourly price forecasts.
  • Real-time streaming pipeline with Kafka KRaft and Spark Structured Streaming successfully generates live trading signals across 3 concurrent queries.
  • Implemented a 24-step recursive prediction engine that loads a saved RF model, receives triggers via Kafka, and produces a full 24h price forecast curve for each currency pair.
  • Built a live Chart.js dashboard that visualizes predicted price trajectories with animated point-by-point rendering and real-time statistics.
  • Stored 96,294 ML predictions in MongoDB via Kafka consumer pipeline, enabling persistent querying across rf_predictions, gbt_predictions, and lr_signals collections.

Future Enhancements
Scope for improvement
  • Deep Learning Integration — Add LSTM or Transformer-based models via Spark DL pipelines for improved time-series prediction.
  • Multi-Node Cluster Deployment — Deploy on a Spark standalone cluster or YARN for true distributed processing across multiple machines.
  • Ensemble Recursive Models — Extend the recursive prediction loop to combine RF + GBT + LSTM predictions for improved multi-model ensemble forecasting.
  • Tree-Based Classifier — Replace Logistic Regression with Random Forest Classifier or XGBoost for better direction prediction.
  • Parquet / Delta Lake — Switch output format from CSV to Parquet or Delta Lake for columnar compression and ACID transactions.
  • Live API Feed — Replace CSV-based Kafka producer with a real forex API (OANDA, Alpha Vantage) for actual live data ingestion.
